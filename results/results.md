# Results

This directory contains all outputs generated by the analysis scripts. **Do not manually create or edit files in this directory.** Every file here should be fully reproducible by running the scripts in `scripts/` in the documented order.

## Directory Organization

```
results/
├── figures/        # Publication-ready figures (.pdf, .png, .svg)
├── model-fits/     # Saved model objects and posterior samples
└── tables/         # Summary tables (.csv, .tex, .html)
```

## Best Practices

### General Principles

- **Scripts generate results; humans do not.** If a result needs to change, change the script — never edit output files by hand. This ensures the scripts remain the single source of truth.
- **All outputs must be regenerable from scratch.** A reviewer should be able to delete this entire directory and recreate its contents by running the analysis pipeline.
- **Version control output selectively.** Commit final, publication-ready outputs (figures, tables) to git so reviewers can inspect them without running the code. Do not commit large model fit objects or raw posterior samples — store those externally and document access.

### Figures (`figures/`)

- **File format:** Save figures in both a vector format (`.pdf` or `.svg`) for publication and a raster format (`.png` at ≥ 300 DPI) for review and supplementary materials.
- **Resolution:** Minimum 300 DPI for print; 600 DPI for figures with fine detail (maps, dense scatter plots).
- **Dimensions:** Match the target journal's column widths. Common widths: single column ~3.5 in, double column ~7 in.
- **File naming:** Use descriptive, lowercase, hyphen-separated names that correspond to the figure number or content:
  - `fig01_study-area-map.pdf`
  - `fig02_population-trend.pdf`
  - `fig-s01_prior-sensitivity.pdf`
- **Fonts:** Embed all fonts in PDF figures. Use sans-serif fonts (Arial, Helvetica, or similar) at ≥ 8 pt for legibility after scaling.
- **Color:** Use colorblind-accessible palettes (e.g., [Okabe-Ito](https://jfly.uni-koeln.de/color/), `viridis`, `ColorBrewer`). Avoid red-green combinations. Check figures in grayscale to ensure they remain interpretable if printed in black and white.
- **Consistency:** Use the same fonts, colors, and line widths across all figures. Define a shared theme or style at the top of the results script and apply it uniformly.
- **Self-containment:** Each figure should be interpretable without referring to the caption. Axis labels should include units; legends should label all elements.

### Tables (`tables/`)

- **File format:** Save tables as `.csv` for data exchange and as `.tex` or `.html` for direct inclusion in manuscripts. Both should be generated by the same script.
- **File naming:** Mirror the figure convention: `table01_model-parameters.csv`, `table-s01_full-covariate-estimates.csv`.
- **Significant figures:** Round to a meaningful number of significant figures — typically 2–3 for parameter estimates and confidence intervals. Avoid reporting more precision than the data justify.
- **Units:** Include units in column headers or a table footnote, not in the data cells.
- **Uncertainty:** Always report a measure of uncertainty alongside point estimates: standard error, credible interval, confidence interval, or standard deviation. Document which measure is reported.

### Model Fits (`model-fits/`)

- **Save the full model object,** not just summaries. Summaries can be recomputed; refitting a model cannot always be avoided but is expensive.
- **File naming:** Include the model name and a date or version stamp so multiple runs can coexist:
  - `model-fit_hierarchical_2024-06-01.rds`
- **MCMC diagnostics:** Save convergence diagnostics alongside the model fit:
  - R-hat values (all chains should converge to R-hat < 1.01)
  - Effective sample size (ESS) for all parameters of interest
  - Trace plots for key parameters
  - Posterior predictive check plots

  - Document their location (shared drive, cloud storage, or data repository).
  - Commit a small summary file (parameter means and credible intervals as `.csv`) for quick inspection.
  - Consider saving samples in a compressed format (`.rds` with compression in R; `.nc` or `.zarr` for Python).

### Linking Outputs to Manuscript

Maintain a clear mapping between output files and manuscript elements. This can be a simple comment in the results script or a table in this file:

| Manuscript element | Output file | Generating script |
|--------------------|-------------|-------------------|
| Figure 1 | `figures/fig01_study-area-map.pdf` | `04_results.R` |
| Figure 2 | `figures/fig02_population-trend.pdf` | `04_results.R` |
| Table 1 | `tables/table01_model-parameters.csv` | `04_results.R` |
| Figure S1 | `figures/fig-s01_prior-sensitivity.pdf` | `03_diagnostics.R` |

This mapping makes it easy for co-authors and reviewers to trace any result back to the code that produced it.
